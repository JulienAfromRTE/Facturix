Règles Métiers Configurables

Gérez les règles de validation conditionnelles qui s'appliquent aux champs de la facture.

ℹ️ Ordre d'application des contrôles
Les contrôles s'appliquent en deux passes successives :
  1. Mapping — chaque champ est contrôlé selon sa définition (obligatoire / type / XPath). C'est la base.
  2. Règles Métiers — les règles ci-dessous s'appliquent après le mapping et prennent le dessus sur les contrôles par défaut.
Exemples de surcharge possibles : rendre un champ obligatoire ou non obligatoire selon la valeur d'un autre champ, imposer une valeur fixe, exiger un signe négatif.

Facturix contrôle les factures électroniques Factur-X / CII en comparant champ par champ la sortie SAP (RDI) et le XML embarqué dans le PDF.

🔍 Contrôle
  • RDI seul — vérifie la présence des champs obligatoires et les règles de gestion
  • XML / PDF seul — contrôle le XML Factur-X embarqué dans le PDF
  • CII direct — analyse un fichier XML CII brut sans PDF
  • RDI vs XML — comparaison complète sortie SAP ↔ sortie Exstream
⚙️ Paramétrage des mappings
  • Ajouter, modifier, supprimer ou cloner des champs BT
  • Configurer le tag RDI, le XPath XML, le caractère obligatoire, la règle de gestion
  • Plusieurs mappings disponibles : CART Simple, CART Groupée, Ventes Diverses, custom
  • Champ "Ignorer les erreurs" pour masquer un BT défaillant connu
📋 Historique des modifications
  • Chaque modification est horodatée et attribuée à un auteur
  • Affichage des valeurs avant / après pour chaque champ modifié
  • Bouton ↩ Revenir pour annuler une modification individuelle
  • Le rollback est lui-même tracé dans l'historique (Rollback de la modification #X)
📐 Règles métiers
  • Toutes les règles sont configurables via l'écran Règles métiers (conditions + actions)
  • 9 règles par défaut : B2G Chorus, avoirs, BT-8=5, client étranger, factures négatives, BT-21-SUR/ISU, etc.
