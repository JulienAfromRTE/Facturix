Règles Métiers Configurables

Gérez les règles de validation conditionnelles qui s'appliquent aux champs de la facture.

ℹ️ Ordre d'application des contrôles
Les contrôles s'appliquent en deux passes successives :
  1. Mapping — chaque champ est contrôlé selon sa définition (obligatoire / type / XPath). C'est la base.
  2. Règles Métiers — les règles ci-dessous s'appliquent après le mapping et prennent le dessus sur les contrôles par défaut.
Exemples de surcharge possibles : rendre un champ obligatoire ou non obligatoire selon la valeur d'un autre champ, imposer une valeur fixe, exiger un signe négatif.

Facturix vérifie la conformité des factures électroniques Factur-X / CII (norme EN 16931). Il peut contrôler le XML embarqué dans un PDF, un fichier RDI issu d'un ERP, ou comparer les deux sources champ par champ.

🚀 Démarrage rapide — quelle option choisir ?

Déposer le fichier PDF (le XML Factur-X est extrait automatiquement).

🔍 Modes de contrôle

XML / PDF seul
Extrait et valide le XML Factur-X embarqué dans le PDF. Vérifie les champs obligatoires, les règles métier, et optionnellement la conformité Schematron EN16931. Aucun fichier RDI nécessaire.
RDI vs XML
Compare champ par champ les données du fichier RDI (sortie ERP, format colonnes fixes cp1252) avec le XML extrait du PDF. Détecte les écarts de valeur, les champs présents d'un côté et absents de l'autre.
CII direct
Analyse un fichier XML CII brut (sans PDF). Utile pour tester des fichiers issus d'une PDP ou générés par un outil tiers. Supporte les namespaces dynamiques.
RDI seul
Contrôle uniquement le fichier RDI : présence des champs obligatoires, cohérence des valeurs, application des règles métier. Utile pour diagnostiquer la sortie ERP avant de générer le PDF.

⚙️ Mapping BT

Le mapping définit, pour chaque champ BT (Business Term de la norme EN 16931) :

  • Le tag RDI attendu dans le fichier ERP (nom de colonne cp1252)
  • Le XPath XML pour localiser la valeur dans le CII
  • Le caractère obligatoire (Oui / Non / Dépendant)
  • La règle de gestion (texte libre, affiché dans les résultats)
  • L'option "Ignorer les erreurs" pour masquer un écart connu

Plusieurs mappings coexistent (ex. facture simple, facture groupée, ventes diverses). Chaque mapping peut être cloné et personnalisé.

📐 Règles métier

Les règles métier ajoutent des contrôles conditionnels au-delà du mapping de base :

  • BT-22-BAR = B2G → rend obligatoires BT-10 (référence acheteur), BT-13 (commande), BT-29 (ID Chorus)
  • BT-3 = 381 (avoir) → rend obligatoires BT-25 (facture d'origine) et BT-26 (date facture d'origine)
  • BT-8 doit valoir "5" (TVA sur les débits)
  • BT-48 ne commence pas par FR → BT-58 (IBAN) devient obligatoire
  • BT-131 négatif → BT-129 (quantité) doit être négatif
  • BT-22-SUR doit valoir "ISU"

Toutes les règles sont éditables depuis l'onglet Règles Métiers, sans modification du code.

📋 Historique & traçabilité

  • Chaque contrôle est enregistré avec date, mode, numéro de facture et score de conformité
  • L'onglet Statistiques affiche les tendances de conformité dans le temps
  • Les modifications de mapping sont auditées : valeur avant / après, auteur, horodatage
  • Bouton ↩ Revenir pour annuler une modification individuelle (le rollback est lui-même tracé)
  • L'onglet Paramétrage permet de purger l'historique par critères (ancienneté, taux de conformité, type)

🔬 Validation Schematron EN16931

En activant le Schematron EN16931 (onglet Paramétrage), Facturix exécute les règles de validation officielles de la norme européenne en plus des contrôles de mapping.

  • Basé sur EN16931-CII v1.3.16 (fichiers XSLT inclus dans schematron/)
  • Chaque erreur Schematron est affichée avec son identifiant de règle (ex. BR-CO-10) et son explication
  • Les erreurs sont distinguées selon leur sévérité (fatal vs warning)

❓ Questions fréquentes

Mon PDF n'a pas de XML embarqué — que faire ?
Un PDF Factur-X contient obligatoirement un fichier XML nommé factur-x.xml (ou xrechnung.xml). Si Facturix indique que le PDF ne contient pas de XML, c'est que le fichier n'est pas un Factur-X valide — vérifiez la source du PDF.
Comment ajouter un nouveau champ BT manquant dans mon mapping ?
Aller dans l'onglet Mapping BT → sélectionner le mapping → cliquer + Ajouter un champ. Renseigner le code BT, le tag RDI correspondant, le XPath XML et le caractère obligatoire. La modification est immédiatement active.
Un champ est en erreur mais c'est normal dans mon cas — comment l'ignorer ?
Dans le mapping, cocher "Ignorer les erreurs" pour ce champ BT. Il continuera d'être contrôlé mais ses erreurs n'impacteront pas le score de conformité et seront affichées en grisé.
Quelle est la différence entre BT-22-BAR, BT-22-SUR, etc. ?
Les champs BT-21/BT-22 apparaissent en paires multiples, chacune identifiée par un code sujet (SubjectCode) : BAR = traitement attendu, SUR = remarques fournisseur, ADN = référence Chorus Pro, AAB = escompte, PMT = indemnité, PMD = pénalités.
La base de données grossit trop — comment nettoyer ?
Aller dans l'onglet Paramétrage → section Purge de l'historique. Filtrer par ancienneté, taux de conformité ou type de facture. Un aperçu (nombre de lignes + espace libéré) est affiché avant confirmation.
